Follow these rules at all times when writing or modifying code:

- Always plan before coding — outline the approach before making changes
- Make small changes only — one thing at a time, no sweeping edits
- Do not refactor outside scope — only touch code directly related to the task
- Keep code simple — prefer the simplest solution that works
- Ask before making large changes — if a change is large or risky, confirm with the user first
